The site search is a small command menu backed by [Pagefind](https://pagefind.app). It gives visitors two search modes: quick navigation for known pages and posts, then full-text search once the query is long enough.

## What Pagefind indexes

Pagefind runs during the Astro build through the [`astro-pagefind`](https://github.com/shishkin/astro-pagefind) integration. The integration creates the index during Astro's `astro:build:done` hook, so the Pagefind files are generated by `astro build`, not by `astro dev` itself.

During development, the integration only serves an existing `dist/pagefind` directory. If you want local dev search to work, build the site once before starting the dev server. Rebuild whenever indexed content changes or after you've run `npm run clean`.

The blog post article body has the `data-pagefind-body` attribute, so Pagefind indexes the actual post content rather than every layout element on the page.

## Opening the search menu

`Search.tsx` renders a search button and a [`cmdk`](https://github.com/dip/cmdk) command dialog. Clicking the button opens the dialog, and the component also registers a global keyboard shortcut:

```tsx
if (event.key.toLowerCase() !== 'k' || (!event.ctrlKey && !event.metaKey)) return
```

Press `Ctrl+K` on Windows and Linux, or `Cmd+K` on macOS, to toggle the search menu.

## Short queries show shortcuts

Before the user types at least two non-whitespace characters, the component does not load Pagefind. Instead, it renders the `pages` and `posts` props that were already passed into the component.

This keeps the initial menu instant. Visitors can open search and jump to a known page or recent post without downloading the Pagefind search bundle.

## Returning results

After Pagefind runs `search(query)`, the component requests data for the first eight results:

```tsx
const data = await Promise.all(search.results.slice(0, 8).map((result) => result.data()))
```

Each result supplies a URL, an HTML excerpt, and optional metadata such as the page title. The UI displays the title, strips HTML tags from the excerpt, and navigates to the result URL when selected.

## Loading and error states

While Pagefind is loading or searching, the command menu shows `Searching...`. If the generated Pagefind bundle is unavailable, the component shows this message:

> Search is unavailable. Try building the site first.

That error is expected in environments where `/pagefind/pagefind.js` has not been generated yet. Run a build first, then start the dev server or deploy the generated static site.
